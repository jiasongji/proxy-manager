1. 概览

Proxy Manager 是一个 Bash 管理脚本，用于部署和维护基于 sing-box 的 AnyTLS、NaiveProxy 与 Shadowsocks 服务。新版支持服务器 A / 服务器 B 双节点：用户连接 A，A 根据规则选择 A 直出或经 B 的 Shadowsocks 落地出站。

项目值

Docker 镜像ghcr.io/sagernet/sing-box:latest

长命令proxy-manager

2. 环境要求

  • Linux 服务器，建议使用 root 用户执行安装。
  • Docker daemon 可用。
  • Docker Compose 可用；缺失时脚本会尝试安装独立二进制。
  • jq 可用：多用户、分流与 JSON 配置管理需要。
  • 可选 grpcurl：用于 V2Ray API stats 用户流量查询。
  • 启用 AnyTLS 或 NaiveProxy 时，需要域名和 TLS 证书/key 文件。
  • 云厂商安全组、面板防火墙需放行实际组件端口。
本手册只保留在目标服务器内执行的安装与运维命令，不展示本地连接或文件传输步骤。

3. 一键安装

3.1 从 GitHub Release 安装

curl -fsSL https://github.com/jiasongji/proxy-manager/releases/latest/download/proxy-manager.sh -o /tmp/proxy-manager.sh
sudo bash /tmp/proxy-manager.sh install

3.2 从 main 分支安装开发版

curl -fsSL https://raw.githubusercontent.com/jiasongji/proxy-manager/main/proxy-manager.sh -o /tmp/proxy-manager.sh
sudo bash /tmp/proxy-manager.sh install

3.3 安装后进入菜单

p-m

4. A/B 拓扑

用户客户端
  │ AnyTLS / NaiveProxy
  ▼
服务器 A（entry_a）
  ├─ split: AI / Google / 自定义规则 → Shadowsocks outbound → 服务器 B
  ├─ split: 其他流量 → direct → 服务器 A 出口
  ├─ all_via_b: 全部流量 → 服务器 B 出口
  └─ all_direct: 全部流量 → 服务器 A 出口

服务器 B（egress_b）
  └─ Shadowsocks inbound → direct 出站

服务器 B 安装示例

p-m install --yes \
  --node-role egress_b \
  --domain b.example.com \
  --server-ip 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>'

服务器 A 安装示例

p-m install --yes \
  --node-role entry_a \
  --domain a.example.com \
  --server-ip 203.0.113.10 \
  --components anytls,naive \
  --cert-file /www/server/panel/vhost/cert/a.example.com/fullchain.pem \
  --key-file /www/server/panel/vhost/cert/a.example.com/privkey.pem \
  --anytls-port 30001 \
  --naive-port 30002 \
  --b-ss-host 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>' \
  --route-mode split
B 的 Shadowsocks 凭据只用于 A 连接 B，不会写入用户客户端配置。建议 B 端口只允许 A 的公网 IP 访问。

6. 多用户管理

用户数据库位于 config/users.json，客户端导出位于 config/client/<user>/。

p-m user list
p-m user add alice --protocols anytls,naive --quota 100GB
p-m user show alice
p-m user disable alice
p-m user enable alice
p-m user passwd alice all
p-m user quota alice unlimited
p-m user reset-usage alice
p-m user export alice
p-m user del alice
  • 禁用用户后，该用户会从 sing-box inbound 用户列表中移除。
  • p-m user export 会显示真实客户端密码，请勿公开粘贴。
  • 从旧单用户版本升级时，旧凭据会迁移为 default 用户。

7. 分流与落地

p-m route show
p-m route mode split
p-m route mode all-via-b
p-m route mode all-direct
p-m route add-domain example.net
p-m route del-domain example.net
p-m route add-keyword example-keyword
p-m route del-keyword example-keyword
模式说明
splitAI / Google / 自定义规则走 B，其余走 A。
all_via_b全部流量走 B，用户看到的出口 IP 为 B。
all_direct全部流量走 A，用于 B 故障时回退。

YouTube 默认不加入 Google 走 B，避免大流量视频消耗 B 带宽；可通过自定义域名按需加入。

8. 流量与限额

p-m stats probe
p-m traffic
p-m traffic alice
p-m quota check
  • 用户流量统计优先使用 sing-box experimental.v2ray_api.stats.users。
  • 需要镜像支持 V2Ray API / gRPC，并且宿主机可用 grpcurl。
  • stats 或 grpcurl 不可用时，流量与限额功能降级提示，不影响代理、分流和用户管理。
  • stats API 默认监听 127.0.0.1:10085，不要暴露到公网。

部署验收

  • B：Shadowsocks landing 启动并限制来源 IP。
  • A：AnyTLS / NaiveProxy 可连接，split 模式命中规则走 B，其余走 A。
  • all_via_b 全部走 B；all_direct 全部走 A。
  • 多用户添加、禁用、导出、改密、删除流程可用。
  • 公开文档不包含真实密码、token、私钥或 B 凭据。

10. 卸载与安全

p-m uninstall
  • 卸载会移除 p-m 与 proxy-manager 命令映射。
  • 默认不删除 Docker、Docker Compose 和 TLS 证书文件。
  • 删除项目目录必须额外输入 DELETE。
  • manager.env、users.json、客户端导出文件均包含敏感信息，应保持 600 权限。
  • 不要公开粘贴 p-m info 或 p-m user export 输出。
