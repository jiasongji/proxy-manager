1. 概览

Proxy Manager 是一个 Bash 管理脚本，用于部署和维护基于 sing-box 的 AnyTLS、NaiveProxy 与 Shadowsocks 服务。新版支持服务器 A / 服务器 B 双节点：用户连接 A，A 根据规则选择 A 直出或经 B 的 Shadowsocks 落地出站。

项目值
GitHub 仓库https://github.com/jiasongji/proxy-manager
Docker 镜像ghcr.io/sagernet/sing-box:latest
短命令p-m
长命令proxy-manager

默认运行镜像仍为上游 ghcr.io/sagernet/sing-box:latest。AI 分流使用 sing-box 新版 route.rule_set 远程 .srs，容器运行环境需要能访问对应规则集 URL。上游 sing-box 更新时，推荐使用 p-m upgrade 进行候选镜像校验和失败回退保护。

1.1 Proxy Lite 轻量副本

proxy-lite/ 是轻量化项目副本，只保留 A/B 中转落地能力：服务器 A 提供 AnyTLS、NaiveProxy 和可选 Shadowsocks 用户入口，并固定经服务器 B 的 Shadowsocks 出站；服务器 B 只提供 Shadowsocks 落地 direct 出口。

2. 环境要求

  • Linux 服务器，建议先切换到 root 用户后执行安装和运维命令。
  • Docker daemon 可用。
  • Docker Compose 可用；缺失时脚本会尝试安装独立二进制。
  • jq 可用：多用户、分流与 JSON 配置管理需要。
  • 可选 grpcurl：用于 V2Ray API stats 用户流量查询。
  • 启用 AnyTLS 或 NaiveProxy 时，需要域名和 TLS 证书/key 文件。
  • 云厂商安全组、面板防火墙需放行实际组件端口。
本手册只保留在目标服务器内执行的安装与运维命令，不展示本地连接或文件传输步骤。

4. A/B 拓扑

用户客户端
  │ AnyTLS / NaiveProxy
  ▼
服务器 A（entry_a）
  ├─ split: AI 远程 rule-set / Google / 自定义规则 → Shadowsocks outbound → 服务器 B
  ├─ split: 其他流量 → direct → 服务器 A 出口
  ├─ all_via_b: 全部流量 → 服务器 B 出口
  └─ all_direct: 全部流量 → 服务器 A 出口

服务器 B（egress_b）
  └─ Shadowsocks inbound → direct 出站

服务器 B 安装示例

p-m install --yes \
  --node-role egress_b \
  --domain b.example.com \
  --server-ip 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>'

服务器 A 安装示例

p-m install --yes \
  --node-role entry_a \
  --domain a.example.com \
  --server-ip 203.0.113.10 \
  --components anytls,naive \
  --cert-file /www/server/panel/vhost/cert/a.example.com/fullchain.pem \
  --key-file /www/server/panel/vhost/cert/a.example.com/privkey.pem \
  --anytls-port 30001 \
  --naive-port 30002 \
  --b-ss-host 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>' \
  --route-mode split
B 的 Shadowsocks 凭据只用于 A 连接 B，不会写入用户客户端配置。建议 B 端口只允许 A 的公网 IP 访问。

5.1 安全升级与失败回退

由于上游 sing-box 会持续更新，生产环境不要只手工拉取镜像后直接重启。推荐使用 p-m upgrade：脚本会先备份当前配置，拉取候选镜像，重新渲染配置，并用候选镜像执行 sing-box check -c；只有校验通过才会应用。候选配置检查失败时不会切换运行容器；重建失败时会恢复更新前快照并尝试重建回退态。

p-m update
p-m backup list
p-m upgrade --image ghcr.io/sagernet/sing-box:latest
p-m check
p-m status

p-m pull-image 只执行 docker pull，不做配置校验、不切换、不重启；如需手动回退，可先查看快照，再恢复指定版本。

p-m backup list
p-m rollback latest
p-m rollback 20260615-120000

6. 多用户管理

用户数据库位于 config/users.json，客户端导出位于 config/client/<user>/。

p-m user list
p-m user add alice --protocols anytls,naive --quota 100GB
p-m user show alice
p-m user disable alice
p-m user enable alice
p-m user passwd alice all
p-m user quota alice unlimited
p-m user reset-usage alice
p-m user export alice
p-m user del alice
  • 禁用用户后，该用户会从 sing-box inbound 用户列表中移除。
  • p-m user export 会显示真实客户端密码，请勿公开粘贴。
  • 从旧单用户版本升级时，旧凭据会迁移为 default 用户。

7. 分流与落地

p-m route show
p-m route mode split
p-m route mode all-via-b
p-m route mode all-direct
p-m route add-domain example.net
p-m route del-domain example.net
p-m route add-keyword example-keyword
p-m route del-keyword example-keyword
模式说明
splitAI 远程规则集、Google、自定义域名或关键词走 B，其余走 A。
all_via_b全部流量走 B，用户看到的出口 IP 为 B。
all_direct全部流量走 A，用于 B 故障时回退。

split 模式下，AI 服务使用 sing-box 新版 route.rule_set 远程 .srs 规则，不使用旧版 geosite / geoip 写法。OpenAI、Claude / Anthropic 单独匹配并排在 AI 总规则前；Claude / Anthropic 使用 anthropic.srs，不要写成 claude.srs。

下面片段可直接粘贴进 sing-box 配置。项目默认 B 出站 tag 为 egress-b；如果你的代理出站 tag 是 proxy，请把 outbound 改成 proxy。如果原配置已有 route.rule_set 和 route.rules，只追加这些条目，不要覆盖原结构。

{
  "route": {
    "rule_set": [
      {
        "tag": "openai",
        "type": "remote",
        "format": "binary",
        "url": "https://raw.githubusercontent.com/MetaCubeX/meta-rules-dat/sing/geo/geosite/openai.srs",
        "update_interval": "1d"
      },
      {
        "tag": "anthropic",
        "type": "remote",
        "format": "binary",
        "url": "https://raw.githubusercontent.com/MetaCubeX/meta-rules-dat/sing/geo/geosite/anthropic.srs",
        "update_interval": "1d"
      },
      {
        "tag": "category-ai-not-cn",
        "type": "remote",
        "format": "binary",
        "url": "https://raw.githubusercontent.com/MetaCubeX/meta-rules-dat/sing/geo/geosite/category-ai-%21cn.srs",
        "update_interval": "1d"
      }
    ],
    "rules": [
      {
        "rule_set": ["openai"],
        "action": "route",
        "outbound": "egress-b"
      },
      {
        "rule_set": ["anthropic"],
        "action": "route",
        "outbound": "egress-b"
      },
      {
        "rule_set": ["category-ai-not-cn"],
        "action": "route",
        "outbound": "egress-b"
      }
    ],
    "final": "direct"
  }
}

Google、可选 YouTube、自定义域名和自定义关键词仍使用内联 domain_suffix / domain_keyword 规则，排在 AI rule-set 规则之后。YouTube 默认不加入 Google 走 B，避免大流量视频消耗 B 带宽；可通过自定义域名按需加入。

8. 流量与限额

p-m stats probe
p-m traffic
p-m traffic alice
p-m quota check
  • 用户流量统计优先使用 sing-box experimental.v2ray_api.stats.users。
  • 需要镜像支持 V2Ray API / gRPC，并且宿主机可用 grpcurl。
  • stats 或 grpcurl 不可用时，流量与限额功能降级提示，不影响代理、分流和用户管理。
  • stats API 默认监听 127.0.0.1:10085，不要暴露到公网。

部署验收

  • B：Shadowsocks landing 启动并限制来源 IP。
  • A：AnyTLS / NaiveProxy 可连接，split 模式命中规则走 B，其余走 A。
  • all_via_b 全部走 B；all_direct 全部走 A。
  • p-m upgrade 会先执行候选镜像配置检查；失败时恢复更新前快照，p-m rollback latest 可手动回退。
  • 多用户添加、禁用、导出、改密、删除流程可用。
  • 公开文档不包含真实密码、token、私钥或 B 凭据。

10. 卸载与安全

p-m uninstall
  • 卸载会移除 p-m 与 proxy-manager 命令映射。
  • 默认不删除 Docker、Docker Compose 和 TLS 证书文件。
  • 删除项目目录必须额外输入 DELETE。
  • manager.env、users.json、客户端导出文件均包含敏感信息，应保持 600 权限。
  • 不要公开粘贴 p-m info 或 p-m user export 输出。
