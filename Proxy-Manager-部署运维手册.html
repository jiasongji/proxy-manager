Proxy Manager 部署运维手册

本手册面向服务器内直接部署：脚本从 GitHub 下载，安装后使用 p-m 或 proxy-manager 管理服务；主菜单回车退出，二级菜单回车返回上一页。

1. 概览 2. 环境要求 3. 一键安装 4. 菜单与命令 5. 组件模式 6. 日常运维 7. Docker 镜像 8. 审计测试 9. 卸载与安全

1. 概览

Proxy Manager 是一个 Bash 管理脚本，用于部署和维护基于 sing-box 的 AnyTLS、NaiveProxy 与 Shadowsocks 落地服务。

项目值
GitHub 仓库https://github.com/jiasongji/proxy-manager
Docker 镜像jiasongji/proxy-manager-sing-box:latest
短命令p-m
长命令proxy-manager

2. 环境要求

  • Linux 服务器，建议使用 root 用户执行安装。
  • Docker daemon 可用。
  • Docker Compose 可用；缺失时脚本会尝试安装独立二进制。
  • 启用 AnyTLS 或 NaiveProxy 时，需要域名和 TLS 证书/key 文件。
  • 云厂商安全组、面板防火墙需放行实际组件端口。
本手册只保留在目标服务器内执行的安装与运维命令，不展示本地连接或文件传输步骤。

3. 一键安装

3.1 从 GitHub Release 安装

curl -fsSL https://github.com/jiasongji/proxy-manager/releases/latest/download/proxy-manager.sh -o /tmp/proxy-manager.sh
sudo bash /tmp/proxy-manager.sh install

3.2 从 main 分支安装开发版

curl -fsSL https://raw.githubusercontent.com/jiasongji/proxy-manager/main/proxy-manager.sh -o /tmp/proxy-manager.sh
sudo bash /tmp/proxy-manager.sh install

3.3 安装后进入菜单

p-m

5. 组件模式

模式说明参数值
全部组件AnyTLS + NaiveProxy + Shadowsocksall
仅 AnyTLS只暴露 AnyTLS 入口anytls
仅 NaiveProxy只暴露 NaiveProxy 入口naive
仅 Shadowsocks作为落地出口ss
组合按需组合启用anytls,ss

参数化安装示例

p-m install --yes \
  --domain example.com \
  --server-ip 203.0.113.10 \
  --components anytls,ss \
  --cert-file /www/server/panel/vhost/cert/example.com/fullchain.pem \
  --key-file /www/server/panel/vhost/cert/example.com/privkey.pem \
  --anytls-port 30001 \
  --ss-port 30003

9. 卸载与安全

p-m uninstall
  • 卸载会移除 p-m 与 proxy-manager 命令映射。
  • 默认不删除 Docker、Docker Compose 和 TLS 证书文件。
  • 删除项目目录必须额外输入 DELETE。
  • 不要公开粘贴 p-m info 输出，因为其中包含真实端口和密码。
