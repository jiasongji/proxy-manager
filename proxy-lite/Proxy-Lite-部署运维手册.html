Proxy Lite 部署运维手册

轻量 A/B 中转落地脚本：服务器 A 提供 AnyTLS / NaiveProxy / 可选 Shadowsocks 用户入口，固定经服务器 B 的 Shadowsocks 出口；使用 PL 或 proxy-lite 管理。

1. 概览 2. 环境要求 3. 安装 4. A/B 拓扑 5. 命令 6. 升级回退 7. 审计 8. 安全

1. 概览

Proxy Lite 是完整 proxy-manager 的轻量化副本，去掉多用户、流量限制和分流功能，只保留 A/B 中转落地闭环。

项目值
短命令PL（大写）
长命令proxy-lite
默认目录/www/wwwroot/<domain>/Proxy-Lite/
默认镜像ghcr.io/sagernet/sing-box:latest
已裁剪：user、route、stats、traffic、quota。如果需要多用户、AI rule-set 分流或限额，请使用完整项目 proxy-manager。

2. 环境要求

  • Linux 服务器，生产运维命令请在 root 用户下执行。
  • Docker daemon 可用。
  • Docker Compose 可用；缺失时脚本会尝试安装独立二进制。
  • jq 可用。
  • 服务器 A 启用 AnyTLS 或 NaiveProxy 时，需要域名和 TLS 证书/key 文件。

3. 安装

3.1 从 GitHub Release 安装

curl -fsSL https://github.com/jiasongji/proxy-lite/releases/latest/download/proxy-lite.sh -o /tmp/proxy-lite.sh
bash /tmp/proxy-lite.sh install

3.2 本仓库内本地验证

bash proxy-lite/proxy-lite.sh help

4. A/B 拓扑

用户客户端
  │ AnyTLS / NaiveProxy
  ▼
服务器 A（entry_a）
  └─ 固定 route.final = egress-b → Shadowsocks outbound → 服务器 B

服务器 B（egress_b）
  └─ Shadowsocks inbound → direct 出站

服务器 B

PL install --yes \
  --node-role egress_b \
  --domain b.example.com \
  --server-ip 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>'

服务器 A

PL install --yes \
  --node-role entry_a \
  --domain a.example.com \
  --server-ip 203.0.113.10 \
  --components anytls,naive \
  --cert-file /path/fullchain.pem \
  --key-file /path/privkey.pem \
  --anytls-port 30001 \
  --naive-port 30002 \
  --b-ss-host 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>'
B 的 Shadowsocks 凭据只作为 A 的上游配置，不进入用户客户端导出。B 端口建议只允许服务器 A 的公网 IP 访问。

7. 审计

bash -n proxy-lite/proxy-lite.sh
shellcheck proxy-lite/proxy-lite.sh
bash proxy-lite/proxy-lite.sh help
bash proxy-lite/proxy-lite.sh backup help
bash proxy-lite/proxy-lite.sh rollback help
bash proxy-lite/proxy-lite.sh upgrade help
  • entry_a 生成 egress-b outbound，route.final 为 egress-b。
  • egress_b 生成 ss-landing-in，route.final 为 direct。
  • 不生成 users.json，不生成 route.rule_set。
  • 客户端导出不包含 B 上游 Shadowsocks 密码。

8. 安全

  • config/lite.env 包含服务凭据和 B 上游凭据，应保持 600 权限。
  • 客户端导出文件包含 A 侧用户入口凭据，不要公开粘贴。
  • 公开文档不得写入真实服务器 IP、域名、端口、密码、token、私钥或证书/key 路径。
  • 卸载默认不删除 Docker、Docker Compose 和 TLS 证书文件。
